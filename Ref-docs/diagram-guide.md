# 다이어그램 가이드 — Mermaid 사용 규칙 및 패턴

## 원칙

- 모든 아키텍처/워크플로우 다이어그램은 **Mermaid** (`\`\`\`mermaid`)로 작성
- ASCII art (`┌─┐│└`) 금지 — GitHub에서 렌더링되지 않음
- 파일 트리(`├──`)와 코드/수식 블록은 Mermaid 변환 대상이 **아님**

## 기본 패턴

```
graph TD (top-down) — 아키텍처, 시스템 구조
graph LR (left-right) — 파이프라인, 워크플로우
```

## 노드 스타일

| 용도 | 문법 | 예시 |
|------|------|------|
| 일반 컴포넌트 | `[Label]` | `UI[Chat UI]` |
| 사용자/외부 | `([Label])` | `User([사용자])` |
| 데이터베이스 | `[(Label)]` | `DB[(SQLite)]` |
| 그룹핑 | `subgraph Name` | `subgraph Core[Core Agent]` |

## 엣지 스타일

| 용도 | 문법 |
|------|------|
| 기본 흐름 | `A --> B` |
| 라벨 포함 | `A -->|라벨| B` |
| 점선 (조건부) | `A -.-> B` |
| 양방향 | `A <--> B` |

## 한/영 규칙

- `README.md` (한국어): 한국어 라벨 사용
- `README_EN.md` (영어): 영어 라벨 사용
- 노드 ID는 동일하게 유지 (라벨만 번역)

## 주의사항

- `subgraph` 이름에 특수문자 사용 시 따옴표로 감싸기
- 노드 라벨에 줄바꿈: `<br/>` 사용
- 복잡한 라벨은 `["큰따옴표"]`로 감싸기
